# StudyOS Google Classroom Backend

This backend provides a secure, read-only Google Classroom integration for StudyOS.

## Features

- Google OAuth 2.0 with only these scopes:
  - `https://www.googleapis.com/auth/classroom.courses.readonly`
  - `https://www.googleapis.com/auth/classroom.coursework.me.readonly`
- Secure token storage in Firestore per user (`user_auth` collection)
- Read-only import endpoint for Classroom assignments
- Assignment normalization to StudyOS format
- OAuth state validation for CSRF protection
- Per-user import rate limiting
- Disconnect endpoint that revokes token + deletes stored credentials

## Endpoints

- `GET /auth/google`
  - Requires `state` query generated by `/auth/google/start`
  - Redirects user to Google consent screen
- `POST /auth/google/start`
  - Requires `Authorization: Bearer <firebase_id_token>`
  - Returns `{ "authUrl": "..." }` for browser-based OAuth launch
- `GET /auth/callback`
  - Exchanges auth code and stores tokens server-side
- `DELETE /auth/google/disconnect`
  - Requires `Authorization: Bearer <firebase_id_token>`
  - Revokes Google token and removes auth record
- `POST /import/google-classroom`
  - Requires `Authorization: Bearer <firebase_id_token>`
  - Optional body:
    ```json
    {
      "existingExternalIds": ["123", "456"]
    }
    ```
  - Returns normalized assignments that are not in `existingExternalIds`

## Normalized Assignment Shape

```json
{
  "externalId": "12345",
  "source": "google_classroom",
  "title": "Essay Draft",
  "className": "English 101",
  "dueDate": "2026-02-10T23:59:00.000Z",
  "notes": "Write 1000 words",
  "estMinutes": 30,
  "importedAt": "2026-02-09T22:00:00.000Z"
}
```

## Setup

1. Copy `.env.example` to `.env`
2. Fill all Google OAuth and Firebase Admin values
3. Install dependencies and run:

```bash
npm install
npm run dev
```

## Security Notes

- Tokens never return to the iOS client
- Logs redact access/refresh tokens
- Backend does not write to Google Classroom and does not persist StudyOS assignments

## V1 API (PostgreSQL-backed)

New scalable endpoints:

- `POST /v1/sync/push`
- `POST /v1/sync/pull`
- `GET /v1/analytics/dashboard?weekStart=YYYY-MM-DD&tz=IANA`
- `GET /v1/analytics/estimated-vs-actual?limit=10`
- `GET /v1/analytics/recent-activity?limit=20`
- `POST /v1/integrations/google-classroom/import`
- `GET /v1/integrations/google-classroom/status`
- `DELETE /v1/integrations/google-classroom/disconnect`

### Required env for V1 API

- `DATABASE_URL` (PostgreSQL connection string)
- `PG_SSL` (`true` or `false`, defaults to `false`)
- `PG_POOL_MAX` (defaults to `20`)

### Initialize schema

```bash
npm run db:migrate
```

### One-time Firestore migration

Migrate legacy Firestore user subcollections (`users/{uid}/assignments`, `users/{uid}/sprints`) into PostgreSQL while preserving sync metadata:

```bash
npm run migrate:firestore
```

Optional flags:

- `--dry-run` to validate without committing
- `--user=<firebaseUid>` to migrate a single user
- `--batch-size=<n>` to tune pagination (default `100`)

Examples:

```bash
npm run migrate:firestore -- --dry-run
npm run migrate:firestore -- --user=abc123
```
